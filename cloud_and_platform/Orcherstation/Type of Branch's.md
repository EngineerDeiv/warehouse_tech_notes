# Environment Flow

```text
DEV → PPT (PRETST) → TST → PPR (PREPRD) → PRD
```

## PRD (Production)

The live environment used by end users and customers.

- Real business traffic
    
- Highest availability requirements
    
- Strict change control
    
- Monitoring and incident management
    

## PPR / PREPRD (Pre-Production)

Environment that closely mirrors Production.

- Final validation before release
    
- Smoke testing
    
- Deployment verification
    
- Performance and integration checks
    

## TST (Testing)

Environment used for functional and integration testing.

- QA validation
    
- User Acceptance Testing (UAT)
    
- Integration testing
    
- Regression testing
    

## PPT / PRETST (Pre-Test)

Intermediate environment between Development and Testing.

- Early validation
    
- Initial deployments
    
- Technical verification
    
- Environment configuration checks
    

## DEV (Development)

Environment where developers build and modify applications.

- Feature development
    
- Unit testing
    
- Debugging
    
- Initial code validation
